CSS Container Queries: The Future of Responsive Design

By Miguel December 5, 2024
Quantum Computing Cryptography Security

Container Queries

Traditional media queries respond to viewport dimensions, but Container Queries allow elements to respond to their immediate container's size. This creates more flexible, reusable components.

Setting Up Container Contexts

To use container queries, you first need to establish a containment context using the container-type property. 

/* Establish containment context */
.card-container {
  container-type: inline-size;
  container-name: card;
}

/* Alternative shorthand */
.card-container {
  container: card / inline-size;
}

Writing Container Queries

Container queries use the @container rule to define responsive behavior based on the container's dimensions. 

/* Default card layout */
.card {
  display: flex;
  flex-direction: column;
  gap: 1rem;
  padding: 1rem;
  border: 1px solid #ddd;
  border-radius: 8px;
}

.card-image {
  width: 100%;
  height: 200px;
  object-fit: cover;
}

.card-content {
  flex: 1;
}

/* When container is wider than 400px */
@container card (min-width: 400px) {
  .card {
    flex-direction: row;
    align-items: center;
  }
  
  .card-image {
    width: 150px;
    height: 150px;
    flex-shrink: 0;
  }
}

/* When container is wider than 600px */
@container card (min-width: 600px) {
  .card {
    padding: 2rem;
  }
  
  .card-image {
    width: 200px;
    height: 200px;
  }
}

Responsive Navigation

Here's how to create a navigation component that adapts to its container size. 

/* Navigation container */
.nav-container {
  container-type: inline-size;
  container-name: navigation;
}

/* Default mobile-first navigation */
.nav {
  display: flex;
  flex-direction: column;
  gap: 0.5rem;
}

.nav-toggle {
  display: block;
  background: none;
  border: 1px solid #333;
  padding: 0.5rem;
  cursor: pointer;
}

.nav-menu {
  display: none;
  flex-direction: column;
  gap: 0.5rem;
}

.nav-menu.active {
  display: flex;
}

/* When container is wide enough for horizontal layout */
@container navigation (min-width: 500px) {
  .nav {
    flex-direction: row;
    align-items: center;
    justify-content: space-between;
  }
  
  .nav-toggle {
    display: none;
  }
  
  .nav-menu {
    display: flex !important;
    flex-direction: row;
    gap: 2rem;
  }
}

Container Query Units

Container queries introduce new units that are relative to the container's dimensions. 

/* Container query length units */
.responsive-text {
  /* 1% of container's inline size (width in horizontal languages) */
  font-size: 4cqi;
  
  /* 1% of container's block size (height in horizontal languages) */
  line-height: 2cqb;
  
  /* 1% of the smaller of cqi and cqb */
  padding: 2cqmin;
  
  /* 1% of the larger of cqi and cqb */
  margin: 1cqmax;
}

Best Practices and Browser Support

Container queries are well-supported in modern browsers. Here are some best practices for implementation:

  • Use progressive enhancement with fallbacks for older browsers
  • Combine container queries with traditional media queries when needed
  • Name your containers for better maintainability
  • Consider performance implications with deeply nested containers

Conclusion

CSS Container Queries revolutionize how we think about responsive design, enabling truly modular components that adapt to their context. As browser support continues to improve, container queries will become an essential tool for modern web development.
